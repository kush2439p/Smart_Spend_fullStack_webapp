REGISTER
- Fields: Name, Email, Password, Confirm Password
- Validate all fields, check email format, passwords match
- If email exists → error: "Email already registered"
- On success → create user with isEmailVerified=false, send verification 
  email, redirect to "Check Your Email" screen (NOT dashboard)

EMAIL VERIFICATION
- Verification link in email opens in browser (normal https link)
- GET /verify-email?token=XXX
- Valid token → mark user as verified → show success page in browser
- Invalid token → show error page
- Expired token → show error with "Request new link" button
- After seeing success page user goes back to app and logs in manually

RESEND VERIFICATION
- Button on "Check Your Email" screen
- POST /api/auth/resend-verification { email }
- If not found → error, if already verified → error
- On success → new token generated, new email sent
- 60 second cooldown on resend button with countdown timer

LOGIN
- Fields: Email, Password
- If not verified → error "Please verify your email first" + resend button
- If wrong credentials → error "Invalid email or password"
- On success → return JWT (24hr expiry) → go to Dashboard

FORGOT PASSWORD
- User enters email → POST /api/auth/forgot-password
- If email not found → error "No account found. Please register first."
- If found → generate reset token (1hr expiry) → send email
- Email contains reset button/link AND the raw token as plain text 
  so user can copy-paste it manually
- After sending show button: "I have a reset token →"

RESET PASSWORD
- Screen has 3 fields:
  1. Reset Token (editable, pre-filled from URL if available)
  2. New Password
  3. Confirm Password
- POST /api/auth/reset-password { token, newPassword }
- Invalid token → error, Expired → error
- Success → clear token from DB → alert "Password reset!" → go to Login

DATABASE USER TABLE
id, name, email, password(hashed), isEmailVerified(default false),
verificationToken, verificationTokenExpiry, resetToken, 
resetTokenExpiry, createdAt, updatedAt

PUBLIC ENDPOINTS (no JWT needed)
POST /api/auth/register
POST /api/auth/login
GET  /verify-email
POST /api/auth/resend-verification
POST /api/auth/forgot-password
POST /api/auth/reset-password

SECURITY
- Always bcrypt passwords, never store plain text
- All tokens are random UUIDs, single use, cleared after use
- Everything else requires valid JWT
